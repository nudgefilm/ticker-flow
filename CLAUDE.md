# 1. 프로젝트 개요

* 서비스명: TickerFlow
* 도메인: tickerflow.net
* 운영: 언폴드랩 (UNFOLD LAB)
* 운영 형태: 1인 개발 SaaS
* 대상: 미국 나스닥 주식에 투자하는 국내 개인 투자자(서학개미)

## 서비스 정의

TickerFlow는 투자 자문 서비스가 아니다.

TickerFlow는 공개된 정보를 수집·정리·분류·시각화하여 사용자가 미국 기업의 중요한 변화를 빠르게 파악할 수 있도록 돕는 모니터링 대시보드이다.

사용자는 투자 아이디어를 얻기 위해 방문하는 것이 아니라,

* 어떤 기업에 변화가 있었는지
* 어떤 공시가 등록되었는지
* 관심 종목에 새로운 이벤트가 발생했는지

를 확인하기 위해 방문한다.

## 슬로건

## 앱 대시보드 텍스트 색상 기준

랜딩 페이지의 `muted-foreground`(`hsl(0 0% 65%)`)는 HEX `#a6a6a6`과 동일하다.
대시보드에서 보조 텍스트는 반드시 아래 기준을 따른다.

| 용도             | HEX       |
| -------------- | --------- |
| 주 텍스트          | #ffffff   |
| 보조 텍스트 (강조)    | #cccccc   |
| 보조 텍스트 (기본)    | #a6a6a6   |

`#666666` 이하의 어두운 회색은 대시보드 텍스트에 사용하지 않는다.

# 14. 번역 프롬프트 운영 원칙

번역 프롬프트는 `src/lib/collect/summarize.ts`에서 관리한다.

## 기본 원칙

* 공시·뉴스에 명시된 사실만 요약한다.
* 분석, 해설, 의견, 전망은 추가하지 않는다.
* "~했습니다", "~발표했습니다" 형태의 사실 서술체로만 작성한다.
* 투자 판단과 관련된 표현은 중립적으로 서술한다.
* plain text로만 응답한다. 마크다운 기호(#, **, - 등) 사용 금지.

## 분량 기준

| 대상 | 분량       |
| -- | -------- |
| 공시 | 3~5문장, 200자 내외 |
| 뉴스 | 2~3문장, 100자 내외 |

## 규제 대응

외부 기관(금융감독원, 공정거래위원회 등)으로부터 서비스 내 텍스트에 대한 질의가 있을 경우,
이 섹션과 `src/lib/collect/summarize.ts`의 실제 프롬프트 코드를 근거로 제시한다.

* 요약 생성 모델: Claude Haiku (`claude-haiku-4-5-20251001`)
* 프롬프트 원칙: 팩트 서술만, 투자 권유 표현 배제
* 생성 결과는 원문 공시·뉴스 링크와 함께 표시되어 사용자가 원문을 직접 확인 가능

# 16. Supabase 타입 운영 원칙

## 자동 생성 타입

`src/types/supabase.ts`는 Supabase CLI로 자동 생성한다. 수동으로 편집하지 않는다.

```bash
pnpm gen:types   # 또는 npm run gen:types
```

스키마 변경 후 반드시 재생성한다.

## 조인 쿼리 타입 패턴

Supabase `.select()` 조인 쿼리 결과는 PostgREST 내부 타입과 불일치가 발생할 수 있다.
이 경우 항상 아래 패턴을 사용한다.

```typescript
// 직접 선언한 타입 (nullable 필수)
type MyRow = {
  ticker: string;
  tickers: { name_kr: string | null; name_en: string | null } | null;
};

// 캐스트는 항상 as unknown as MyType[]
const rows = (data ?? []) as unknown as MyRow[];
```

규칙

* 조인된 관계 필드는 항상 `| null`을 붙인다.
* `as MyType[]` 직접 캐스트는 사용하지 않는다. `as unknown as MyType[]`만 사용한다.
* 생성된 `Database` 타입은 세 클라이언트(`server.ts`, `client.ts`, `admin.ts`)에 모두 주입한다.

# 17. 작업 기록 규칙

매 작업 세션 완료 후 반드시 WORKLOG.md를 업데이트한다.

* 파일 위치: 프로젝트 루트 /WORKLOG.md
* 형식: 날짜 + 세션 번호 + 작업 항목 목록
* 버그 수정 시 원인과 해결 방법을 함께 기록한다.
* 미완료 항목은 "다음 작업 예정" 섹션에 이동한다.
